For DATA 305, I built a GitHub Pages site that includes a home page, a resume page, and a blog.

### Why I chose Jekyll
Jekyll is simple, works well with GitHub Pages, and makes it easy to write blog posts in Markdown while keeping the design consistent.

### What I learned
- Small layout components (like a shared navigation bar) make multi-page sites way easier.
- A little CSS goes a long way — especially for mobile responsiveness.
- Getting `baseurl` right is the difference between “everything works” and “every link is broken.”
